# ACCEPTANCE｜人员管理可见性控制

## 1. 验收标准

### A. 人员管理
- [x] 列表增加“显示”开关列。
- [x] 切换开关可实时保存状态。
- [x] 默认新建人员状态为“显示”。

### B. 新建/编辑项目
- [x] 在“创建项目”弹窗中，负责人下拉列表**仅显示**可见人员。
- [x] （隐含）已有项目的负责人在编辑时若被隐藏，不影响回显（逻辑已处理，若当前负责人ID匹配则显示）。

### C. 看板与图例
- [x] 左侧图例（人员看板/产品线设置）不显示隐藏的人员。
- [x] 时间轴泳道中，由隐藏人员负责的项目**不显示**。
- [x] 项目统计数（如各产品线项目总数）应排除隐藏项目（在 Swimlane 中已过滤）。

## 2. 执行结果

- **测试时间**: 2026-01-19
- **测试环境**: 本地开发环境 (Mac/Chrome)
- **测试过程**:
    1.  进入人员管理，关闭某人员（秦晋）的显示开关。
    2.  确认“新建项目”下拉框中无该人员。
    3.  确认时间轴上的项目数量减少，该人员负责的项目消失。
    4.  重新开启开关，确认项目和选项恢复。
- **结论**: 功能符合预期，验证通过。
